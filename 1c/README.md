# HTTP-сервис 1С:УНФ для приёма звонков из Plusofon connector

Этот каталог содержит описание и псевдокод для создания HTTP-сервиса в конфигурации 1С:УНФ.

## Что нужно сделать в конфигураторе

### 1. Опубликовать базу на веб-сервере

Сервис → Публикация на веб-сервере. Убедитесь что флаг **«Публиковать HTTP-сервисы»** включён.

### 2. Создать HTTP-сервис

- **Имя**: `ПлюсофонСервис`
- **Корневой URL**: `plusofon`
- Добавьте шаблон URL: `/calls` — метод **POST**
- Добавьте шаблон URL: `/quickcall` — метод **POST** (опционально, для обратной отладки)

Полный адрес после публикации:
```
POST http(s)://your-1c-host/unf/hs/plusofon/calls
```

### 3. Обработчик метода POST /calls

Скопируйте код из файла `1c/PlusofonService.bsl` в обработчик метода.

### 4. Пользователь для синхронизации

Создайте технического пользователя (например, `plusofon_sync`) с минимальными правами:
- Роль: базовые права
- Дополнительно: право на запись Лидов, Взаимодействий, чтение Контрагентов/Контактов

Логин и пароль этого пользователя укажите в `.env` коннектора (`UNF_USER`, `UNF_PASSWORD`).

## Структура входящего JSON (от коннектора)

```json
{
  "connect_time": "2025-11-01 14:32:00",
  "number_a":    "79161234567",
  "number_b":    "74991234567",
  "cld":         "74991234567",
  "direction":   "external",
  "duration":    142,
  "account":     "manager@company",
  "record_url":  "https://api.plusofon.ru/.../record.mp3",
  "source":      "plusofon"
}
```

| Поле | Описание |
|---|---|
| `connect_time` | Дата и время начала разговора |
| `number_a` | Номер звонящего (A-сторона) |
| `number_b` | Номер принимающего (B-сторона) |
| `cld` | Вызываемый номер (может отличаться от number_b при переадресации) |
| `direction` | `external` — внешний звонок, `internal` — внутренний |
| `duration` | Длительность разговора в секундах |
| `account` | Логин/аккаунт сотрудника в Plusofon |
| `record_url` | Ссылка на запись разговора (если есть) |
| `source` | Всегда `"plusofon"` |
