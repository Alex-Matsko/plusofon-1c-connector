# plusofon-1c-connector

Полинг-коннектор для интеграции **Plusofon (облачная АТС)** с **1С:УНФ (Управление нашей фирмой)**.

Работает без вебхуков — периодически опрашивает Plusofon API и записывает новые звонки в 1С через HTTP-сервис.
Также предоставляет endpoint для инициации исходящего звонка из 1С (click-to-call).

## Возможности

- 🔄 Автоматическая синхронизация звонков из Plusofon → 1С:УНФ
- 📞 Создание лида при входящем с неизвестного номера
- 📋 История звонков во Взаимодействиях 1С
- 🔗 Ссылка на запись разговора в карточке звонка
- 📲 Исходящий звонок из 1С (click-to-call через Plusofon quickcall API)
- 🐳 Docker-ready, state сохраняется между рестартами

## Быстрый старт

### 1. Клонируйте и настройте окружение

```bash
git clone https://github.com/Alex-Matsko/plusofon-1c-connector.git
cd plusofon-1c-connector
cp .env.example .env
# Отредактируйте .env — заполните токены и адрес 1С
```

### 2. Запустите через Docker Compose

```bash
docker-compose up -d
```

Проверьте что сервис живой:
```bash
curl http://localhost:3000/health
```

### 3. Настройте 1С:УНФ

См. подробную инструкцию в [`1c/README.md`](./1c/README.md).  
Код HTTP-сервиса — в [`1c/PlusofonService.bsl`](./1c/PlusofonService.bsl).

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|---|---|
| `PLUSOFON_TOKEN` | ✅ | Bearer-токен из ЛК Plusofon: Еще → Разработчикам → API |
| `PLUSOFON_CLIENT_ID` | ✅ | Client ID из того же раздела |
| `UNF_URL` | ✅ | Адрес опубликованной базы 1С, напр. `http://192.168.1.100/unf` |
| `UNF_USER` | ✅ | Логин технического пользователя 1С |
| `UNF_PASSWORD` | ✅ | Пароль технического пользователя 1С |
| `POLL_INTERVAL_SEC` | ➖ | Интервал опроса в секундах (по умолчанию `30`) |
| `PORT` | ➖ | Порт HTTP-сервера коннектора (по умолчанию `3000`) |
| `API_SECRET` | ➖ | Секрет для защиты `/quickcall` (заголовок `X-Api-Key`) |
| `STATE_FILE` | ➖ | Путь к файлу состояния (по умолчанию `/data/state.json`) |

## Как работает поллер

1. При старте читает `lastSync` из `/data/state.json` (по умолчанию — 24 часа назад).
2. Каждые `POLL_INTERVAL_SEC` секунд делает `GET /api/v1/call` с параметрами `date_from/date_to`.
3. Для каждого звонка вычисляет ключ дедупликации и пропускает уже обработанные.
4. Новые звонки отправляет `POST` в 1С (`/hs/plusofon/calls`).
5. При успехе помечает звонок как обработанный и обновляет `lastSync`.

## Исходящий звонок из 1С

Чтобы позвонить клиенту прямо из карточки 1С, добавьте в форму клиента/лида кнопку,  
которая делает HTTP POST на коннектор:

```http
POST http://connector-host:3000/quickcall
Content-Type: application/json
X-Api-Key: <API_SECRET>

{
  "number":      "79161234567",
  "line_number": "79161112233"
}
```

- `number` — номер клиента
- `line_number` — номер/SIP менеджера (Plusofon сначала позвонит менеджеру, потом соединит с клиентом)

## Ограничения

- **Нет real-time уведомлений** при входящем звонке (webhooks у вас не подключены) — задержка синхронизации равна `POLL_INTERVAL_SEC`.
- Имена реквизитов `ВзаимодействиеТелефонныйЗвонок`, `Лиды` и т.д. в BSL-коде могут отличаться в вашей версии УНФ — проверьте через Конфигуратор.
- Ссылки на записи разговоров доступны только если Plusofon их передаёт в статистике вызовов (зависит от тарифа).
