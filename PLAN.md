# Пошаговый план интеграции Plusofon ↔ 1С:УНФ

## Архитектура

```
Plusofon API
    │
    │ polling (каждые 30 с)
    ▼
[Node.js connector :8080]
    │  POST /hs/plusofon/calls
    ▼
[1С:УНФ HTTP-сервис]
    │
    ▼
Документ «Событие» (тип: ТелефонныйЗвонок)
```

Кнопка «Позвонить» в карточке клиента 1С:
```
1С → POST :8080/quickcall → Plusofon API /call/quickcall
```

---

## Шаг 1. Node.js коннектор (папка `src/`)

### 1.1 `src/index.js`
- Читает `PORT` из `.env` (по умолчанию **8080**, не 3000)
- Запускает Express-сервер и poller

### 1.2 `src/poller.js`
- Каждые `POLL_INTERVAL_SEC` секунд запрашивает `GET /api/v1/call` у Plusofon
- Параметры: `date_from`, `date_to`, `direction=all`, нужные `columns[]`
- Дедуплицирует по ключу `connect_time|account|number_a|number_b` (хранится в `state.json`)
- Для каждого нового звонка вызывает `POST /hs/plusofon/calls` в 1С

### 1.3 `src/server.js`
- `GET /health` — проверка жизни контейнера
- `POST /calls` — проксирует payload поллера в 1С:УНФ (внутренний endpoint)
- `POST /quickcall` — принимает запрос из 1С и вызывает Plusofon `/call/quickcall`
  - Защищён заголовком `X-Api-Key: <API_SECRET>`

### 1.4 `.env` (скопировать из `.env.example`)
```
PLUSOFON_TOKEN=...
PLUSOFON_CLIENT_ID=...
UNF_URL=http://<адрес-1С>/unf
UNF_USER=plusofon_sync
UNF_PASSWORD=...
POLL_INTERVAL_SEC=30
PORT=8080
API_SECRET=<случайная строка>
STATE_FILE=/data/state.json
```

## Шаг 2. HTTP-сервис в 1С:УНФ (расширение конфигурации)

### 2.1 Создать HTTP-сервис
- Имя: `Plusofon`
- Корневой URL: `/hs/plusofon`
- Шаблон URL: `/calls`
- Метод: `POST`
- Обработчик: код из `1c/PlusofonService.bsl`

### 2.2 Что делает обработчик (файл `1c/PlusofonService.bsl`)
1. Парсит JSON-тело запроса от коннектора
2. Определяет направление звонка (входящий / исходящий)
3. Нормализует внешний номер (убирает `+`, `-`, пробелы, снимает 8/7)
4. Ищет Контрагента / Контактное лицо по номеру через `РегистрСведений.КонтактнаяИнформация`
5. Ищет пользователя 1С по полю `account` из Plusofon
6. Создаёт документ **Событие** с `ТипСобытия = ТелефонныйЗвонок`:
   - `Тема`, `Содержание`, `НачалоСобытия`, `ОкончаниеСобытия`
   - `ВходящееИсходящееСобытие`, `Состояние = Завершено`
   - Табличная часть `Контакты`: номер телефона + найденный контакт
   - `Автор` — найденный пользователь
   - В `Содержании` — длительность и ссылка на запись разговора
7. Возвращает `{"ok": true, "event_ref": "..."}`

### 2.3 Технический пользователь
- Создать пользователя `plusofon_sync` в 1С
- Права: **HTTP-сервис Plusofon** + чтение справочников + создание документов Событие
- Аутентификация: Basic (логин/пароль из `.env`)

## Шаг 3. Кнопка «Позвонить» в 1С

### 3.1 Форма карточки Контрагента (расширение)
- Добавить команду `ПозвонитьЧерезPlusofon`
- В обработчике команды:

```bsl
// Клиентский модуль формы
Процедура ПозвонитьЧерезPlusofon(Команда)
	Номер = ПолучитьНомерДляЗвонка(); // выбрать из контактной информации
	Если НЕ ЗначениеЗаполнено(Номер) Тогда
		Предупреждение("Не задан номер телефона");
		Возврат;
	КонецЕсли;

	// Получить SIP/внутренний номер текущего пользователя
	// (хранится в реквизите «ПлюсофонЛиния» справочника Пользователи)
	Линия = ПолучитьЛиниюТекущегоПользователя();

	Параметры = Новый Структура;
	Параметры.Вставить("number", Номер);
	Параметры.Вставить("line_number", Линия);

	// Отправить на коннектор
	СтрокаПараметров = СформироватьJSON(Параметры);
	Соединение = Новый HTTPСоединение("<IP-коннектора>", 8080,,,,,);
	ЗапросHTTP = Новый HTTPЗапрос("/quickcall");
	ЗапросHTTP.Заголовки["Content-Type"] = "application/json";
	ЗапросHTTP.Заголовки["X-Api-Key"] = "<API_SECRET>";
	ЗапросHTTP.УстановитьТелоИзСтроки(СтрокаПараметров, КодировкаТекста.UTF8);
	Ответ = Соединение.ОтправитьДляОбработки(ЗапросHTTP);
	Если Ответ.КодСостояния = 200 Тогда
		ПоказатьОповещениеПользователя("Звонок инициирован", , Номер);
	Иначе
		Предупреждение("Ошибка инициирования звонка: " + Ответ.ПолучитьТелоКакСтроку());
	КонецЕсли;
КонецПроцедуры
```

## Шаг 4. Docker-запуск

```bash
cp .env.example .env
# Заполнить .env реальными значениями
docker compose up -d
docker compose logs -f connector
```

Проверить:
```bash
curl http://localhost:8080/health
# → {"status":"ok","time":"..."}
```

---

## Шаг 5. Проверка сквозного сценария

1. Позвоните на номер компании
2. Через 30 с (или раньше, если уменьшить `POLL_INTERVAL_SEC`) коннектор увидит звонок
3. В 1С откройте журнал документов «Событие» — должна появиться запись с типом «Телефонный звонок»
4. Убедитесь, что контрагент заполнен, автор определён, длительность и ссылка на запись присутствуют

## Файловая структура репозитория

```
plusofon-1c-connector/
├── .env.example          # шаблон переменных окружения (PORT=8080)
├── docker-compose.yml    # порт 8080:8080
├── Dockerfile
├── package.json
├── PLAN.md               # этот файл — план реализации
├── src/
│   ├── index.js          # точка входа, порт 8080
│   ├── server.js         # Express: /health, /calls, /quickcall
│   ├── poller.js         # опрос Plusofon API, отправка в 1С
│   ├── db.js             # state.json: last_sync + seen keys
│   ├── logger.js         # winston-логгер
│   └── utils.js          # normPhone и прочие утилиты
└── 1c/
    ├── PlusofonService.bsl  # обработчик HTTP-сервиса для УНФ
    └── README.md            # инструкция по установке в 1С
```
